## Objective

Build the core React components for the chat interface: ChatMessage for displaying messages, MessageInput for user input, and ChatContainer as the main layout that orchestrates the chat UI.

## Context

This is Sprint 2 Stream B of the AI Chat Web Interface project. The frontend scaffolding (WU-S1B-001) is complete with a working React + TypeScript setup using Vite. We now need to build the chat UI components that will later integrate with the backend API (Sprint 3).

## Requirements

- Determine component hierarchy and data flow
- Define props interfaces for each component
- Plan state management approach
- Ensure components are reusable and maintainable

### 2. ChatMessage Component

- Support different styling for user vs assistant messages

- Clean layout with proper spacing

## Design Decisions

### Component Hierarchy
```
ChatContainer
├── ChatMessage (multiple, one per message)
└── MessageInput
```

### TypeScript Interfaces
```typescript
// Message type
interface Message {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  timestamp: Date;
}

// ChatMessage props
interface ChatMessageProps {
  message: Message;
  isLoading?: boolean;
}

// MessageInput props
interface MessageInputProps {
  onSendMessage: (content: string) => void;
  disabled?: boolean;
}

// ChatContainer state
interface ChatContainerState {
  messages: Message[];
  isLoading: boolean;
  error: string | null;
}
```

### State Management
- ChatContainer manages all state (messages, loading, errors)
- MessageInput is controlled component (no internal state)
- ChatMessage is presentational component (props only)

### Styling Approach
- CSS Modules for scoped styles
- User messages: right-aligned, blue background
- Assistant messages: left-aligned, gray background
- Loading indicator: subtle animation
- Error messages: red border/background
