# Work Unit: Backend API Development

**ID**: WU-S2A-001
**Sprint**: Sprint 2 - Stream A (Backend API Development)
**Created**: 2025-11-09
**Status**: PLANNED

---

## Objective

Create a lightweight FastAPI backend that provides a REST API for the chat interface to communicate with the llm_caller_cli module. The API should handle chat requests, integrate with the LLM module, and enforce localhost-only security.

---

## Context

**Dependencies**:
- Requires WU-V27-001 (Module Integration) to be complete
- llm_caller_cli module must be available and working at `src/llm_caller_cli/`
- Frontend (WU-S1B-001) will consume this API

**Integration Points**:
- Frontend chat interface will POST messages to `/chat` endpoint
- Backend will use llm_caller_cli to call LM Studio
- Response flows back through API to frontend

**Security Model**:
- Localhost-only (bind to 127.0.0.1)
- Reject external requests
- No authentication needed (localhost trust model)

---

## Scope

### In Scope

1. **FastAPI Framework Setup**
   - Initialize FastAPI application
   - Configure CORS for localhost origins
   - Set up server to bind to 127.0.0.1 only
   - Add health check endpoint

2. **API Design**
   - POST /chat endpoint for message handling
   - Request/response schemas using Pydantic
   - Error response schema
   - OpenAPI documentation (auto-generated)

3. **LLM Integration**
   - Import and configure llm_caller_cli
   - Create service layer for LLM calls
   - Handle llm_caller_cli exceptions
   - Map LLM responses to API responses

4. **Error Handling**
   - Request validation errors (422)
   - LLM service errors (500/503)
   - Configuration errors (500)
   - External request rejection (403)

5. **Testing**
   - Unit tests for endpoints
   - Unit tests for service layer
   - Integration tests with mock LLM
   - Integration test with real LM Studio
   - 100% code coverage

6. **Documentation**
   - API documentation (OpenAPI/Swagger UI)
   - README with setup instructions
   - Example requests/responses

### Out of Scope

- Authentication/authorization (localhost trust model)
- Database persistence (stateless API)
- Rate limiting (single-user localhost)
- Frontend implementation (separate work unit)
- Advanced features (streaming, multi-turn, etc.)

## Design

### Architecture

```
Frontend (Browser)
    ↓ HTTP POST /chat
FastAPI Backend (localhost:8000)
    ↓ Python call
llm_caller_cli Module
    ↓ HTTP POST
LM Studio (localhost:1234)
```

### File Structure

```
src/api/
├── __init__.py
├── main.py              # FastAPI app, startup/shutdown
├── routes.py            # Endpoint definitions
├── schemas.py           # Pydantic request/response models
├── service.py           # LLM service layer (calls llm_caller_cli)
└── middleware.py        # Security middleware (localhost-only)

tests/api/
├── __init__.py
├── test_routes.py       # Endpoint tests
├── test_service.py      # Service layer tests
├── test_middleware.py   # Security tests
└── test_integration.py  # Full API integration tests

docs/
└── API.md               # API documentation and examples
```

### API Schema

**POST /chat**
```json
Request:
{
  "message": "string (required, 1-10000 chars)"
}

Success Response (200):
{
  "response": "string",
  "model": "string",
  "timestamp": "ISO8601"
}

Error Response (4xx/5xx):
{
  "error": "string",
  "detail": "string (optional)"
}
```

**GET /health**
```json
Response (200):
{
  "status": "healthy",
  "llm_available": true/false
}
```

### Implementation Approach

1. **main.py**: Create FastAPI app, add middleware, include routers
2. **schemas.py**: Define Pydantic models for validation
3. **middleware.py**: Implement localhost-only check
4. **service.py**: Wrap llm_caller_cli with error handling
5. **routes.py**: Implement endpoints using service layer
6. **tests/**: Write comprehensive test suite

### Error Handling Strategy

- **ValidationError** → 422 with field details
- **LLM timeout** → 503 Service Unavailable
- **LLM error** → 500 Internal Server Error with sanitized message
- **External request** → 403 Forbidden
- **Configuration error** → 500 with safe message

## Risk Assessment

**Low Risk**:
- FastAPI is mature and well-documented
- llm_caller_cli is already tested and working
- Localhost-only reduces security complexity

**Medium Risk**:
- LLM service availability (mitigated by health check)
- Error message sanitization (mitigated by explicit error mapping)

**Mitigation**:
- Comprehensive error handling for all LLM failure modes
- Health check endpoint to verify LLM availability
- Clear error messages for debugging
