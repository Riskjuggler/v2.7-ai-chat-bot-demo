## Risks & Mitigations

### Risk 1: E2E Tests Flaky
**Probability: Medium | Impact: Medium**

**Mitigation:**
- Use Playwright's auto-wait features
- Add explicit waits for API responses
- Use data-testid attributes for reliable selectors
- Mock LLM for consistent responses in automated tests

### Risk 2: Performance Target Not Met
**Probability: Low | Impact: Medium**

**Mitigation:**
- LM Studio with local models should be fast (<1s)
- Network latency minimal (localhost)
- If needed, optimize API client (connection pooling)
- Document actual performance, adjust target if needed

### Risk 3: LM Studio Not Available
**Probability: Low | Impact: High**

**Mitigation:**
- User can install LM Studio (free, easy)
- E2E tests use mock LLM (don't require real LLM)
- Manual testing can use OpenAI/Anthropic as fallback
- Document LM Studio setup in TESTING.md

### Risk 4: CORS Issues
**Probability: Low | Impact: Medium**

**Mitigation:**
- Backend already configured for localhost CORS
- Test CORS configuration during security validation
- Document CORS settings in INTEGRATION.md
- Provide troubleshooting guide
